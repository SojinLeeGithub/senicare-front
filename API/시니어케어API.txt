로그인(front -> back) 

입력 (요청)
- 아이디(userId)		문자열	필수
- 비밀번호(password) 	문자열	필수

1. 유효성 검사
	
2. 입력한 아이디 비밀번호가 데이터베이스에 존재하는지?
2 - 1. 아이디로 데이터베이스 조회
만약 존재하지 않는 아이디 - (로그인 정보 불일치)
2 - 2. 비밀번호 비교
만약 일치하지 않는 비밀번호 - (로그인 정보 불일치)

3. 토근 발행

출력(응답)
- 성공(
	코드 문자열 필수 (code),
	코드설명 문자열 필수 (message)
	토큰 문자열 필수 (accessToken)
	유효기간 숫자(정수) 필수 (expires)
	)

- 데이터 유효성 검사 실패(
	코드 문자열 필수 (code),
	코드설명 문자열 필수 (message)
	토큰 문자열 필수 (accessToken)
	유효기간 숫자(정수) 필수 (expires)
	)
- 로그인 정보 불일치(
	코드 문자열 필수 (code),
	코드설명 문자열 필수 (message)
	토큰 문자열 필수 (accessToken)
	유효기간 숫자(정수) 필수 (expires)
	)	
- 데이터베이스 에러(
	코드 문자열 필수 (code),
	코드설명 문자열 필수 (message)
	토큰 문자열 필수 (accessToken)
	유효기간 숫자(정수) 필수 (expires)
	)

<아이디 중복확인>

입력 (요청)
-아이디 문자열 필수 

절차 (로직)
1. 유효성 검사
2. 입력한 아이디가 존재하는지?
2 - 1 데이터베이스에 아이디 조회
만약 존재하는 아이디가 있다면 (중복됨)

출력 (응답)
-성공 (중복되지 않음)
-유효성 검사 실패
-실패 (중복됨)


전화번호 인증

입력 (요청) 
- 전화번호 문자열 필수 숫자로만 구성 길이제한 (11자리)

절차 (로직)
1. 유효성 검사

2. 입력한 전화번호에 코드 발송

2 - 1. 중복되는 전화번호인지 확인
만약 중복된다면 (이미 존재하는 전화번호)

2 - 2. 코드 생성

2 - 3. 전화번호로 코드 전송
만약 전송에 실패한다면 (코드 전송 실패)

2 - 4. 데이터베이스에 전화번호와 코드를 저장
(데이터베이스 에러)


출력 (응답)
-성공
-유효성 검사 실패
-이미 존재하는 전화번호
-코드 전송 실패
-데이터베이스 에러

인증확인

입력 (요청)
-인증번호 문자열 필수
-

1. 유효성 검사

2. 전화번호와 인증번호로 데이터베이스 조회
존재하지 않는다면 (인증번호 일치하지 않음)

출력 (응답)
-성공
-유효성 검사 실패
-데이터베이스 에러
-인증번호 불일치
